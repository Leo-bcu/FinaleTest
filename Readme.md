## 阶段一：数据基建与安全治理 (直击细则 1、2、3，共 30 分)
这个阶段是整个项目的地基，主要解决“数据不够”和“质量不高”的问题。

对应任务：

【需求1/细则1】确定痛点：物流链路不透明、静态调度脆弱。使用 Supply Chain Logistics 关联数据集。

【需求2/细则2】数据增强（化劣势为满分）： 针对 OrderList.csv 只有 9215 行的短板，采用 AI 核密度估计（KDE）或分布采样算法，自动生成长尾订单，补齐并突破 1 万条大关。

【需求3/细则3】AI 治理与安全： 摒弃规则清洗。采用 孤立森林 (Isolation Forest) 算法剔除重量与运费逻辑冲突的异常订单（质量管理）；采用 加盐哈希 (Salted Hash) 对 Customer ID 和 Order ID 进行不可逆加密脱敏（安全治理）。

任务标签： 数据处理、算法清洗、数据库 (SQLite) 入库。

## 阶段二：智能分析与运筹决策 (直击细则 4、5，共 20 分)
这个阶段要求系统不仅能画图表，还得能结合大模型给出具有业务价值的“指挥命令”。

对应任务：

【需求4/细则4】BI 与大模型融合： 基于 SQLite 数据库搭建前端看板（展现各港口吞吐量、平均运输耗时等）。通过 Text-to-SQL 技术，让大模型能够根据人类自然语言直接查询并生成物流统计结论。

【细则5】模型解释与决策（重点拿分）： 进行变量探索（如：容量上限、运费成本）。引入 SHAP 模型解释技术，让大模型分析出影响延误的关键特征（如：“分析表明，中转天数增加的主因是仓储容量饱和”），并结合 WhCapacities（容量限制）提出合理的改道分流决策。

任务标签： 数据可视化、大模型 API 调用、特征重要性分析。

## 阶段三：知识发现与图谱问答 (直击细则 6、7，共 20 分)
这个阶段利用现成的高质量结构表，极速完成图谱构建与 RAG 问答打通。

对应任务：

【需求5/细则6】知识图谱构建： 利用 PlantPorts、ProductsPerPlant、VmiCustomers 三张表，直接抽取“工厂-港口”、“工厂-产品”、“工厂-特供客户”三元组，构建包含几百个实体和关系的供应链图谱。包括本体概念定义与图谱质量评估。

【需求5/细则7】图谱增强问答 (RAG)： 将上述图谱拓扑与规则库（容量、成本）结合。当提问“A仓库爆仓，产品X该发往哪？”时，系统能沿着图谱路径搜索替代方案并给出准确回答。

任务标签： 知识图谱 (NetworkX/Neo4j)、大模型 RAG 链路。

## 阶段四：核心优化与实验验证 (直击需求 6/细则 8，共 10 分)
这是组长你用来展现深度逻辑、拉开与其他组差距的“降维打击”模块。

对应任务：

【需求6/细则8】有效改进与创新：

自反思纠错 Text-to-SQL： 针对大模型生成 SQL 易报错的问题，引入错误捕获与自我 Debug 的闭环机制。

属性动态衰减图谱： 针对静态图谱无法应对突发拥堵的痛点，引入根据 BI 看板动态更新路线权重的机制；同时在底层设置 deterministic=False 增强模型鲁棒性。

【需求6/细则8】验证效果： 设计消融实验，通过对比数据（如 SQL 一次性执行成功率、复杂拥堵环境下的召回率）证明优化的有效性。

任务标签： Agentic Workflow、对比实验设计、算法微调。
