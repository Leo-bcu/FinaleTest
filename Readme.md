## 选题二：金融风控大数据的智能信贷分析与反欺诈系统

### 1. 数据集与痛点

- **推荐数据：** Kaggle 经典 *Home Credit Default Risk（信贷违约风险数据集）*，包含数十万条真实信贷表格数据。
- **业务痛点：** 欺诈样本极少，传统机器学习容易“漏报”；风控模型往往是“黑盒”，信贷员无法向客户合理解释被拒绝贷款的真实原因。

### 2. 完整的 5 步实施路径

- **Step 1：AI 驱动的数据准备与质量治理（基础分）**

  - **质量治理：** 使用孤立森林（Isolation Forest）等无监督算法自动筛查并清洗掉“收入负数”等异常脏数据。
- **Step 2：BI 与大模型深度融合及可解释性分析（基础分）**
  - 可视化呈现不同行业、不同信贷周期的逾期违约率走势图。
  - 将风控预测模型（如 LightGBM）与大模型对接，将枯燥的算法输出转化为业务人员看得懂的决策依据。

- **Step 4：核心创新点落地：多模态融合风控与 SHAP 可解释性翻译（满分加分项）**

### 3. 本选题专项【创新点细化】
💡 创新点（1）：基于 SHAP（沙普利值）特征可解释性的大模型“白话翻译”引擎传统做法： 传统 AI 风控只给出一个 0 到 1 之间的概率分数，用户和信贷员完全不知道为什么被拒。我们的优化： 引入机器学习可解释性工具 SHAP，计算出当前申请人各项特征（如：负债比、逾期次数、工作年限）对风险分数的绝对贡献值。大模型读取这些 SHAP 权重后，自动组织语言生成一份《个性化风控拒贷解释报告》：“该用户被拒绝，主因是其近期负债收入比超过 65% 的警戒线，且近 3 个月内有异地频繁申贷行为。”💡 创新点（2）：表格与文本“多模态融合”风控架构传统做法： 只看数字表格，忽略了用户填写的贷款申请陈述、客服沟通日志等文本。我们的优化： 采用双塔机制，利用文本编码器提取申请陈述的语义 Embedding，与表格特征进行融合，共同输入风控分类器。实验证明，加入非结构化文本多模态特征后，模型对复杂边缘样本的识别能力大幅提升。
- **消融实验（Ablation Study）设计：**
对比 `仅表格特征模型`、`表格+文本多模态融合模型` 的 $AUC$ 和 $KS$ 指标。通过表格展示多模态融合带来的风控精度提升。

### 3. 本选题专项【创新点细化】
💡 创新点（1）：基于时变权重的动态图谱增强 RAG（Dynamic Graph RAG）传统做法： 图谱是死板不动的，大模型检索出来的永远是固定的物理路线。我们的优化： 让知识图谱中路段边（Edges）的权重（通行时间）随着 BI 系统计算出的实时路况动态刷新。当交警提问“某干道发生连环追尾，周边如何调度”时，大模型在时变动态图谱上进行实时路径搜索，输出基于当前最前沿路况的动态避堵调度方案。💡 创新点（2）：引入适度随机性增强复杂路况泛化鲁棒性（底层训练优化）传统做法： 追求极致的确定性，导致模型在面对突发恶劣天气（如暴雨）时严重过拟合。我们的优化： 为了打破模型在单一正常场景下的过拟合，在底层时序预测模型的训练配置中，主动修改设定，强制引入适度的随机性（如设置 deterministic=False）。实验证明，这种针对底层参数的调整，使得模型在极端噪声或突发拥堵数据下的泛化预测能力和鲁棒性反而得到了显著提升。
- **消融实验（Ablation Study）设计：**
测试在“晴天（常态）”和“暴雨（极端异常）”两种环境下，开启与关闭随机性优化（`deterministic=True` vs `False`）时，路况预测的均方误差（$MSE$）对比，用数据证明底层优化对系统泛化性的贡献。
